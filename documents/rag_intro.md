# RAG 检索增强生成技术概述

## 什么是 RAG？

检索增强生成（Retrieval-Augmented Generation，简称 RAG）是一种结合检索系统和大型语言模型（LLM）的技术架构。

RAG 的核心思想是：当 LLM 需要回答问题或生成内容时，先从外部知识库中检索相关信息，然后将这些信息作为上下文提供给 LLM，帮助其生成更准确、更可靠的答案。

## RAG 的工作流程

1. **索引阶段（Indexing）**
   - 文档加载：从各种来源（PDF、网页、数据库）加载文档
   - 文本分块：将长文档分割成较小的片段
   - 向量化：使用 embedding 模型将文本转换为向量
   - 存储：将向量存储到向量数据库中

2. **检索阶段（Retrieval）**
   - 将用户问题转换为向量
   - 在向量数据库中搜索最相似的文档片段
   - 返回 top-k 个最相关的文档

3. **生成阶段（Generation）**
   - 将检索到的文档与用户问题组合成 prompt
   - 发送给 LLM 生成最终答案

## RAG 的优势

- **时效性**：可以访问最新信息，无需重新训练模型
- **可解释性**：答案可以追溯到具体来源
- **成本效益**：比微调模型更经济
- **灵活性**：可以轻松更新知识库

## RAG 的应用场景

- 企业内部知识库问答
- 客服机器人
- 文档摘要和问答
- 医疗、法律等专业领域问答系统
